First Game Engine
Demo
Project Source Code

2012-05-07

Mac OSX
Xcode 4
G++, GNU Make
C++, OpenGL, SDL

A 3D platformer game engine created in OpenGL, SDL, and C++. Simple collision detection is done by flattening circle and square collisions in the xz plane. generally, if the y-value of 2 objects are at a level where the 2 objects can interact, the game engine first tests an outter bounding circle for collisions and then (if passed) an inner bounding square for collisions. a base actor class was created and all objects are instantiations of derived classes from the base actor class.

A demo of the program can be found here. The demo first navigates a test map used to debug the game engine as I developed it and second navigates through my current iteration of the game engine (simple static models, textures, and a map level included).

Simple Particle System
Demo
Project Source Code

2012-04-14

Mac OSX
Xcode 4
G++, GNU Make
C++, OpenGL, GLUT

A simple particle system that simulates a spewing teapot using OpenGL pixels. Particles fade and change color from blue to white as time passes. Simple particle-to-ground collision is enabled. A particle will be reproduced at the teapot's spout at the end of its life (faded out or collision with floor).

A demo of the program can be found here. The demo navigates around the spewing teapot and toggles the number of particles emitted from the teapot's spout.

Extremely Fast File System
effs_v3.tar.gz

2011-12-02

Ubuntu 10.04
JEdit
GCC, GNU Make
C

The Extremely Fast File System (EFFS) simulates the file system of a hard disk. The lower layer driver of the file system is "fisk.h". "effs.c" consists of the main code base of the file system and accesses the "fisk.h" API. The test harness "effs_test.c" and interactive shell "shell.c" sit above "effs.c" and access its API. EFFS is based on the old Unix File System that uses 12 direct pointers to data and then single, double, and triple levels of indirection. EFFS consists of a superblock, inode space, and data space.

I have included a readme that specifies a few simplifying assumptions and outlines how to use the EFFS shell. I have also included a report written by Jaime Lai and Nazma Panjwani which goes into further detail of the file system's design.

Random Movie Selector
recursiveRename.rar
selectRandomMovie.rar

2011-07-13

Windows
IDLE
Python 2.7.2

I watch a lot of movies at home with my girlfriend. Unfortunately, we often can't decide what to watch. I created two simple python scripts to solve our dilemma: recursiveRename.py and selectRandomMovie.py.

Description of recursiveRename.py:
Recursively walk through each folder and file starting from the current directory and rename any of '(', ')', ' ' to '.' so that selectRandomMovie.py will work properly.

Description of selectRandomMovie.py:
Randomly selects and plays a movie from the current directory. The current directory must contain only sub-directories with a playable movie in the first sub-directory level.
