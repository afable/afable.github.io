Shoal
Website
Read Report
Project Source Code

2014-04-04

Web, Shoal
Squid Cache
Tornado
Javascript

We sought to improve upon Shoal's current Web.py framework in terms of performance and scalability. Specifically, we aimed to increase the number of requests the server can handle per second and decrease the average time it takes to do geographical distance calculations. We also wanted to improve the feel of the web interface to allow less knowledgeable clients to see what Shoal is doing. In order to achieve the aforementioned goals, the following approach was taken:

1. Employed a new Python framework tailored for performance. The Tornado Python web server was considered as a replacement for the current Web.py web server.

2. Implemented a new algorithm and utilized caching to speed up computationally expensive operations, among which was the geographical distance calculation.

3. Implemented a new web interface for tracking proxy servers using Ajax and the JavaScript library DataDriven Documents, D3.js.

Healthy Tweets
Website
Project Source Code

2013-10-25

Web, Twitter
Python
Google APIs
Map Reduce

There are a lot of opinions and research denoting a correlation between self awareness and well-being. Healthy Tweets explores how much the level of self consciousness actually relates to self-health in modern society. If we analyze tweet data of people who are always tweeting about worrying of how fit they are or about their diet, can we correlate those findings with census data on clinical visits, number of unhealthy diagnosis, mental clinic populations, or medical costs per province? Can we contrast negative vs positive self images and their effect to possible health issues? These are some of the many questions we would like to answer.

First Game Engine
Demo
Project Source Code

2012-05-07

Mac OSX
Xcode 4
G++, GNU Make
C++, OpenGL, SDL

A 3D platformer game engine created in OpenGL, SDL, and C++. Simple collision detection is done by flattening circle and square collisions in the xz plane. generally, if the y-value of 2 objects are at a level where the 2 objects can interact, the game engine first tests an outter bounding circle for collisions and then (if passed) an inner bounding square for collisions. a base actor class was created and all objects are instantiations of derived classes from the base actor class.

A demo of the program can be found here. The demo first navigates a test map used to debug the game engine as I developed it and second navigates through my current iteration of the game engine (simple static models, textures, and a map level included).

Extremely Fast File System
Read Report Project Source Code

2011-12-02

Ubuntu 10.04
JEdit
GCC, GNU Make
C

The Extremely Fast File System (EFFS) simulates the file system of a hard disk. The lower layer driver of the file system is "fisk.h". "effs.c" consists of the main code base of the file system and accesses the "fisk.h" API. The test harness "effs_test.c" and interactive shell "shell.c" sit above "effs.c" and access its API. EFFS is based on the old Unix File System that uses 12 direct pointers to data and then single, double, and triple levels of indirection. EFFS consists of a superblock, inode space, and data space.

I have included a readme that specifies a few simplifying assumptions and outlines how to use the EFFS shell. I have also included a report written by Jaime Lai and Nazma Panjwani which goes into further detail of the file system's design.

Random Movie Selector
Project Source Code

2011-07-13

Windows
IDLE
Python 2.7.2

I watch a lot of movies at home with my girlfriend. Unfortunately, we often can't decide what to watch. I created two simple python scripts to solve our dilemma: recursiveRename.py and selectRandomMovie.py.

recursiveRename.py:
Recursively walk through each folder and file starting from the current directory and rename any of "(", ")", and spaces to periods so that selectRandomMovie.py will work properly.

selectRandomMovie.py:
Randomly selects and plays a movie from the current directory. The current directory must contain only sub-directories with a playable movie in the first sub-directory level.
